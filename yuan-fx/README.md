# 🇨🇳→🇹🇭 Yuan FX Timing — เครื่องมือจับจังหวะแลกบาท→หยวน สำหรับผู้นำเข้า

> เป้าหมาย: ช่วย **ผู้นำเข้าที่จ่ายซัพพลายเออร์จีนเป็นเงินหยวน (CNY) แบบประจำสม่ำเสมอ**
> ตัดสินใจว่า "งวดนี้ควรแลกมาก / น้อย / รอ" เพื่อลดต้นทุนค่าเงิน — **ไม่ใช่บอทเก็งกำไร**

## ทำไมไม่ใช่ "บอทเทรด"?
คุณไม่ได้เก็งกำไรค่าเงิน คุณแค่ต้อง "ซื้อหยวน" เป็นระยะเพื่อจ่ายของ ดังนั้นกลยุทธ์ที่เหมาะคือ
**DCA (เฉลี่ยต้นทุน) + เอียงน้ำหนักตามความถูก/แพง** ไม่ใช่การเข้า-ออกเก็งกำไรแบบเทรดเดอร์
งานวิจัยชี้ชัดว่า การพยากรณ์ค่าเงินระยะสั้นแทบไม่ชนะ random walk — แต่ผู้นำเข้า "ได้เปรียบ"
ตรงที่ **ยืดหยุ่นเรื่องจังหวะได้** จึงควรใช้ความได้เปรียบนั้นแทนการเดาทิศ

## วิธีใช้ Pine Script บน TradingView
1. เปิด [TradingView](https://www.tradingview.com) → ช่องค้นสัญลักษณ์พิมพ์ **`CNYTHB`** (เช่น `FX_IDC:CNYTHB`) → ตั้ง Timeframe = **1D (รายวัน)**
2. เมนูล่าง **Pine Editor** → วางโค้ดจาก `pine/cnythb_import_timing.pine` ทั้งหมด → **Add to chart**
3. กดไอคอน ⚙️ ที่ชื่ออินดิเคเตอร์เพื่อปรับ:
   - **งบปกติต่อรอบ (บาท)** = งบที่ตั้งใจแลกต่อหนึ่งงวด
   - **KBank TT Sell CNY** = กรอกเรท "ขาย" หยวนของกสิกรวันนี้ (ดูจาก [หน้า FX กสิกร](https://www.kasikornbank.com/th/rate/Pages/Foreign-Exchange.aspx)) เพื่อเห็นส่วนต่างที่แบงก์คิด
4. ตั้ง **Alert**: คลิกขวาบนกราฟ → Add alert → เลือกเงื่อนไข เช่น *"หยวนถูกมาก (Z<-1.5)"* → รับแจ้งเตือนเข้าแอป/อีเมล/ไลน์

### อ่านค่ายังไง
ราคา CNYTHB = **บาทต่อหยวน 1 หน่วย** → **ค่ายิ่งต่ำ = หยวนยิ่งถูก = ดีต่อคุณ**

| โซน (Z-score) | ความหมาย | ตัวคูณงบแนะนำ |
|---|---|---|
| Z ≤ −1.5 | หยวนถูกมาก | **x2.0** (front-load งวดหน้าด้วย) |
| −1.5 < Z ≤ −0.5 | ถูกกว่าค่าเฉลี่ย | x1.3 |
| −0.5 < Z < +0.5 | ปกติ | x1.0 (ตามแผน DCA) |
| +0.5 ≤ Z < +1.5 | แพง | x0.7 (แลกเท่าที่จำเป็น) |
| Z ≥ +1.5 | แพงมาก | x0.4 (แลกขั้นต่ำ รอ) |

> ⚠️ **ข้อควรระวัง:** อย่าหยุดแลกทั้งหมดเพื่อรอให้ถูกสุด — ความเสี่ยงคือหยวนแข็งขึ้นแล้วคุณจ่ายแพงกว่าเดิม
> โมเดลนี้ "เอียงน้ำหนัก" ไม่ใช่ "เปิด/ปิด" จึงควรแลกขั้นต่ำเสมอแม้ในโซนแพง เพื่อกันความเสี่ยง
