# 📐 ตรรกะโมเดล: DCA + Tilt สำหรับผู้นำเข้าจ่ายหยวนประจำ

## 1. ทำไมต้องเป็น DCA + Tilt (ไม่ใช่การพยากรณ์ทิศ)

งานวิจัยอัตราแลกเปลี่ยนระยะสั้นเกือบทั้งหมดสรุปตรงกันว่า **โมเดลพยากรณ์แทบไม่ชนะ
"random walk"** (Meese–Rogoff puzzle, 1983 เป็นต้นมา) โดยเฉพาะกับสกุลที่ถูกแทรกแซง
อย่างหยวน ที่ PBOC คุมผ่าน "central parity fixing" รายวัน + กรอบ ±2%

แต่ **ผู้นำเข้าไม่จำเป็นต้องทำนายทิศให้แม่น** — ข้อได้เปรียบของคุณคือ **ความยืดหยุ่นเรื่องจังหวะ**:
จ่ายเร็วขึ้น/ช้าลงได้บ้าง, แลกเผื่อล่วงหน้าได้, แบ่งจ่ายได้ จึงควรใช้กลยุทธ์ที่:

1. **DCA (Dollar-Cost Averaging)** — แลกสม่ำเสมอเป็นฐาน เพื่อ "เฉลี่ยต้นทุน" และกันความเสี่ยงพลาดจังหวะ
2. **Tilt (เอียงน้ำหนัก)** — เมื่อหยวน "ถูกผิดปกติเทียบค่าเฉลี่ย" ก็แลกมากขึ้น/เผื่อล่วงหน้า; เมื่อ "แพงผิดปกติ" ก็แลกแค่ขั้นต่ำที่ต้องใช้

นี่คือ **mean-reversion เชิงต้นทุน** ไม่ใช่การเก็งกำไร

---

## 2. ตัวชี้วัดที่ใช้

| ตัวชี้วัด | สูตร | ตีความ |
|---|---|---|
| **ราคายุติธรรม (basis)** | `SMA(CNYTHB, 120)` | ค่าเฉลี่ยราคา ~ครึ่งปีทำการ = "ราคากลาง" อ้างอิง |
| **Z-score** | `(price − basis) / stdev(price,120)` | วัดว่าราคาห่างค่าเฉลี่ยกี่ "ส่วนเบี่ยงเบนมาตรฐาน" — ติดลบ = หยวนถูก |
| **Percentile rank** | อันดับราคาปัจจุบันในรอบ 120 วัน (0–100) | 0% = ถูกสุดในรอบ, 100% = แพงสุด |
| **RSI(14)** | โมเมนตัม | ยืนยันว่าหยวนกำลังอ่อน/แข็ง (เสริม ไม่ใช่ตัวหลัก) |

> ทำไมใช้ z-score? เพราะมัน **normalize** ความผันผวน — ช่วงที่ค่าเงินผันผวนสูง (เช่นสงครามการค้า)
> เกณฑ์ ±1.5σ จะกว้างขึ้นเองโดยอัตโนมัติ ไม่เหวี่ยงสัญญาณถี่เกินไป

---

## 3. ตารางการตัดสินใจ (Decision Table)

| Z-score | สถานะ | การกระทำ | ตัวคูณงบ (tilt) |
|---|---|---|---|
| `Z ≤ −1.5` | หยวนถูกมาก | แลกเพิ่ม + front-load งวดหน้า | **2.0×** |
| `−1.5 < Z ≤ −0.5` | ถูกกว่าเฉลี่ย | แลกปกติ + เพิ่มเล็กน้อย | 1.3× |
| `−0.5 < Z < +0.5` | ปกติ | แลกตามแผน DCA | 1.0× |
| `+0.5 ≤ Z < +1.5` | แพง | แลกเท่าที่จำเป็น / เลื่อน | 0.7× |
| `Z ≥ +1.5` | แพงมาก | แลกขั้นต่ำ รอจังหวะ | 0.4× |

**ตัวอย่าง:** งบปกติ 100,000 บาท/งวด → ถ้า Z = −1.8 (หยวนถูกมาก) ระบบแนะนำแลก ~200,000 บาท
(งวดนี้ + เผื่องวดหน้า) ; ถ้า Z = +1.6 (แพงมาก) แนะนำแลกแค่ ~40,000 บาท เท่าที่ต้องจ่ายจริง

> 💡 **กฎเหล็ก:** ไม่ว่าจะแพงแค่ไหน **ห้ามแลก 0** ถ้ามีบิลต้องจ่าย — ความเสี่ยงที่แท้จริง
> ของผู้นำเข้าคือ "รอแล้วหยวนแข็งขึ้น" ไม่ใช่ "แลกแล้วหยวนอ่อนลง"

## 4. การปรับจูน (Tuning)
- **maLen / zLen**: 120 วัน (~ครึ่งปี) เหมาะกับการจ่ายรายเดือน. ถ้าจ่ายถี่ (รายสัปดาห์) ลองสั้นลง 60;
  ถ้าจ่ายไม่บ่อย (รายไตรมาส) ลองยาว 200–250
- **เกณฑ์ Z (1.5 / 0.5)**: ขยับให้ไวขึ้น (1.0 / 0.3) ถ้าต้องการสัญญาณบ่อย; แคบลงถ้าอยากเลือกเฉพาะจังหวะเด่น
- **เทียบเรทกสิกร**: ช่อง KBank TT Sell ทำให้เห็น **spread ของแบงก์** (ปกติ ~1.5–3% เหนือ mid-market)
  ถ้า spread กว้างผิดปกติ อาจคุยขอเรตพิเศษ (K GLOBAL TRADE) หรือเทียบ Super Rich/โอนผ่านบริการ FX อื่น

## 5. ข้อจำกัด & ส่วนขยายในอนาคต
- โมเดลนี้เป็น **rule-based** ล้วน — โปร่งใส ตรวจสอบได้ ไม่ overfit
- **ส่วนขยายที่เป็นไปได้** (ดู `data_sources_api.md`):
  - เพิ่มเลเยอร์ **ข่าว/sentiment** (NLP จับพาดหัวข่าว PBOC/trade war) เป็น "ตัวกรองความเสี่ยง"
    เช่น ถ้ามีข่าวลบแรง ๆ ให้ระงับการ front-load
  - ดึง **USDCNH + USDTHB** มาประกอบ เพื่อแยกว่าหยวนอ่อนเพราะ "หยวนเอง" หรือเพราะ "ดอลลาร์/บาท"
  - Backtest ย้อนหลังว่ากลยุทธ์ tilt ประหยัดต้นทุนได้กี่ % เทียบกับ DCA ล้วน
